## Lessons Learned

### Process
- **Small Task Chunks**: Breaking Phase 3 into 3 distinct implementation chunks (Environment, Core Logic, Testing) allowed for easier debugging and validation.
- **Spec-Driven Consistency**: Maintaining the `task.md` and `BUILD_REPORT` artifacts in real-time prevented "drift" between the implementation and the original design.

### Technical
- **LangGraph State Management**: Discovered that `StateGraph(PBIPState)` strictly filters keys. Ensuring `schema_yaml` was explicitly in the state definition resolved critical "missing key" errors.
- **TMDL for Human-Readable Models**: Using TMDL instead of binary PBIX files simplified the generation logic using Jinja2 templates.
- **M-Expression Shadowing**: Inline `#table()` literals provided an excellent "zero-config" rendering experience for the user.
- **Professional CLI**: Choosing `Typer` + `Rich` significantly improved the developer experience with status spinners and formatted logs.

### Communication
- **Agent Banners**: The `> [!IMPORTANT] Invoking Specialist` banners provided clear traceability for which specialist was handling which logic.
- **Visual Recaps**: Using `project_recap.html` to summarize complex technical decisions (like Sequential vs Observer patterns) helped bridge the gap between code and design.
